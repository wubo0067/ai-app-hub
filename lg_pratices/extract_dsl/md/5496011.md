## Resolution

- We are aware that some faulty CPU or memory sometimes causes a single bit-flip.
- Please contact the hardware vendor, perform complete hardware diags and as-needed replacement.

## Root Cause

- The immediate cause of the crash was a hard lockup.
- A hard lockupis defined as a bug that causes a CPU to loop in kernel mode for more than 10 seconds with interrupts disabled.
- In this case, it's highly possible that a single bit-flip on a sighand address due to some CPU/Memory fault caused a issue.
- A task on a CPU on which the hard lockup occurred was looping in a siglock spinlock on the sighand struct in kernel mode indefinitely with interrupts disabled.
