## Environment

- Red Hat Enterprise Linux 6.9
- Red Hat Enterprise MRG Realtime 2.5
- kernel-rt-3.10.0-327.rt56.198.el6rt


## Issue

System panic due to a hard lockup in interrupt context caused by deadlock on rt_mutex->wait_lock


## Resolution

This issue has been resolved byRHSA-2017:3295.

Please update tokernel-rt-3.10.0-693.11.1.rt56.597.el6rt.x86_64or later.


## Root Cause

This bug can be observed when some task contends for a rt_mutex->wait_lock, which might be also needed in some interrupt context.

The problem is that when the task "enqueues" itself for the wait_lock, blocking any other contenders, then there's a short window (< 1 nanosecond), where it might get interrupted by an interrupt which would also need the same wait_lock, creating a deadlock.
