- Please update the system tokernel-5.14.0-362.8.1.el9_3or later which contains the fix for this bug

## Root Cause

- A bug innvmedriver was resulting in significant time required fordma_pagelookup indma_pool->page-list. The excessive time indma_pagelookup resulted in a delay in releasing thedma_pool->lockspin lock which resulted in hard lockup errors and hang. Please refer to the notes inDiagnostic Stepsfor more information about the same.

`nvme``dma_page``dma_pool->page-list``dma_page``dma_pool->lock`
