## Resolution

- This issue has been fixed in kernel-3.10.0-1160.el7 viaRHSA-2020:4060.


### Workaround

- As a current workaround, boot the system withiommu=pton the kernel command line.

`iommu=pt`
## Root Cause

- iovaallocation from 32bit address range is getting stuck oniova_rbtree_lock

`iova``iova_rbtree_lock`
