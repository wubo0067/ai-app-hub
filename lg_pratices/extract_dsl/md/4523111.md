## Resolution

- Please upgrade the kernel to the latest
- Please consider upgrading microcode_ctl to the latest
- Please engage the server manufacturer and run complete hardware diagnostics


## Root Cause

- Unfortunately we are not definitely sure of the exact cause of the hard LOCKUPs.
- If the issue still persists after all the suggested things mentioned inResolutionsection have been done, please open a support case with Red Hat support and provide some more additional vmcore files for us to take a look to investigate the issue further.

Bit 9 is not set on CPU 55. The Interrupt flag (IF) is a system flag bit which is bit 9 of the FLAGS register, may be set or cleared by programs with sufficient privileges, as usually determined by the operating system. If the flag is set to 1, maskable hardware interrupts will be handled. If cleared (set to 0), such interrupts will be ignored. IF does not affect the handling of non-maskable interrupts (NMIs) or software interrupts generated by the INT instruction.

On the other hand bit 9 (interrupts are enabled) is set on CPU 29 where the hard LOCKUP is met. hard LOCKUP is met when CPU spents longer period of time in the same code with interrupts disabled. But interrupts are enabled on this CPU at the time of the hard LOCKUP.
