System's current configuration regarding kernel message is shown in the below which is print every messages to the console.


```
kernel.printk = 7   4   1   7
```

`kernel.printk = 7   4   1   7`The default for kernel.printk is '4 4 1 7' which is printing only warning or more urgents messages to the console. The current setting in the system is asking to print even debugging messages to be printed on console as well as save in /var/log/messages. As writing to console is slower than writing to disk, it can cause of hard lockup when there are massive output to handle. Please consider to change kernel.printk to have '4 4 1 7' to avoid this in the future.
