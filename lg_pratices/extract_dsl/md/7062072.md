## Environment

- Red Hat Enterprise Linux 8
- VMware VM


## Issue

- systemd-sysctl.service is emitting the following errors:


```
Mar 23 11:28:38 localhost systemd[1]: Starting Apply Kernel Variables...
Mar 23 11:28:38 localhost systemd-sysctl[921]: Couldn't write '0' to 'net/ipv6/conf/default/accept_redirects', ignoring: No such file or directory
Mar 23 11:28:38 localhost systemd-sysctl[921]: Couldn't write '0' to 'net/ipv6/conf/all/accept_redirects', ignoring: No such file or directory
Mar 23 11:28:38 localhost systemd-sysctl[921]: Couldn't write '0' to 'kernel/nmi_watchdog': Unknown error 524
Mar 23 11:28:38 localhost systemd[1]: systemd-sysctl.service: Main process exited, code=exited, status=1/FAILURE
Mar 23 11:28:38 localhost systemd[1]: systemd-sysctl.service: Failed with result 'exit-code'.
Mar 23 11:28:38 localhost systemd[1]: Failed to start Apply Kernel Variables.
```

`Mar 23 11:28:38 localhost systemd[1]: Starting Apply Kernel Variables...
Mar 23 11:28:38 localhost systemd-sysctl[921]: Couldn't write '0' to 'net/ipv6/conf/default/accept_redirects', ignoring: No such file or directory

## Resolution

- The following entries should be removed:


```
/etc/sysctl.d/50-libreswan.conf:net.ipv6.conf.default.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv6.conf.all.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv4.conf.default.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv4.conf.all.accept_redirects = 0
```

`/etc/sysctl.d/50-libreswan.conf:net.ipv6.conf.default.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv6.conf.all.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv4.conf.default.accept_redirects = 0
/etc/sysctl.d/50-libreswan.conf:net.ipv4.conf.all.accept_redirects = 0`- The following entry should be removed:


```
/etc/sysctl.conf:kernel.nmi_watchdog = 0
```

`/etc/sysctl.conf:kernel.nmi_watchdog = 0`
## Root Cause

- Since IPv6 is disabled with "ipv6.disable=1" added to the cmdline, the corresponding proc interfaces do not exist.
