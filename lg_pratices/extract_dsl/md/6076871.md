## Resolution

- It is advisable to contact the hardware vendor, run complete hardware diagnostics, and upgrade the BIOS and the FW to the latest to eliminate the possibility of some faulty hardware preventing the CPUs from executing or updating times properly.


## Root Cause

- A hard lockup is defined as a bug that causes a CPU to loop in kernel mode without giving other tasks a chance to run (with interrupts being disabled) for more than 10 seconds.
- We could hardly avoid suspecting the possibility of some faulty hardware preventing the CPUs from executing or updating times properly because nothing is indicative of some kernel bug that was causing the lockups where almost all CPUs, on which the lockups occurred, were just running the idle threads with interrupts being enabled.
