- The issue is already fixed in InfoScale 7.4.1 Patch 1200. Refer following url for patch details.

## Root Cause

- Vxnetdgets into soft lockup when port scan tool keeps sending packets over the network.
- Vxnetdacquires spin lock on cpu while polling for packet without giving up on cpu. so if the packets are kept sending vxnetd may get into soft lockup status.

A soft lockup is the symptom of a task or kernel thread using and *not* releasing a CPU 
for a longer period of time than allowed. 

crash> runq -T | grep 'CPU 0'
  CPU 0: 10017.71 secs

The default soft lockup threshold is twice the amount of time the hard lockup threshold.

crash> watchdog_thresh
watchdog_thresh = $1 = 10

crash> pd (watchdog_thresh*2)
$2 = 20
