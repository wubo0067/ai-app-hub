## Environment

- Red Hat Enterprise Linux (RHEL)
- 3rd-party antivirus software
- hardware TPM module


## Issue

The system experiences hard lockup while running antivirus software that scans TPM-related files under the/sysfolder.

`/sys`
## Resolution

Please consider working with the antivirus software vendor in order to address this issue.

Meanwhile, if the software supports some form of ignore list, consider using it to exclude/sysfolder from scanning.

`/sys`
## Root Cause

Antivirus software may traverse/sysfolder during the system scan and access the following files:

`/sys`- /sys/devices/pnp0/00:05/tpm/tpm0/ppi/vs_operations
- /sys/devices/pnp0/00:05/tpm/tpm0/ppi/tcg_operations

`/sys/devices/pnp0/00:05/tpm/tpm0/ppi/vs_operations``/sys/devices/pnp0/00:05/tpm/tpm0/ppi/tcg_operations`This causes an unintentional effect of kernel entering a long-term ACPI call chain that may trigger hard lockup watchdog.
