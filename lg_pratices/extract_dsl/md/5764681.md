## Root Cause

- A root cause is unknown at this time.
- Multiple hard lockups occurred on multiple CPUs on which tasks were running different code paths that had nothing to do with each other.
- Lots of MCE events were being logged and also hrtimer/perf interrupts were taking too long time to be processed during the time frame of the occurrence of the soft lockup followed by the multiple hard lockups.
- It may be possible that some faulty hardware was causing some CPUs to be getting slow or stalled on which tasks were processing some jobs with zone->lock or rq->lock spinlocks being held hence the soft/hard lockups.
