## Environment

Red Hat Enterprise Linux 8.


## Issue

The server crashed due to hard lockup which was caused by a deadlock between code running on CPU 2 and CPU 30.


```
[110459.790039] Kernel panic - not syncing: Hard LOCKUP
[110459.790039] CPU: 30 PID: 592228 Comm: qemu-kvm Kdump: loaded Tainted: G           OE    --------- -  - 4.18.0-193.el8.x86_64 #1
```

`[110459.790039] Kernel panic - not syncing: Hard LOCKUP
[110459.790039] CPU: 30 PID: 592228 Comm: qemu-kvm Kdump: loaded Tainted: G           OE    --------- -  - 4.18.0-193.el8.x86_64 #1`
## Resolution

This bug is being tracked in private Bugzilla1813666


## Root Cause

- Issue occurs because of the dead lock between kernel code due to wrong ordering of spinlocks.
