# Named-chroot.service causes soft lockup, hard lockup or OOM when it was restarted or stopped.

## Environment

- Red Hat Enterprise Linux 7.4
- bind-chroot-9.9.4-50.el7.x86_64
- bind-9.9.4-50.el7.x86_64


## Issue

- Trigger a soft lockup, hard lockup or OOM in /var/log/messages when executing "systemctl restart or stop named-chroot.service".
- Named-chroot.service causes memory leaks and kernel panics.


## Resolution

- Update bind and bind-chroot packages to 9.9.4-61.el7 or later.


## Root Cause

- This happens because of two things.  First, the setting of /var/named/chroot/var/named as a private mount and not a shared mount only affects the original namespace it was mounted in.  It's still shared in any other namespaces.  Second, since it's a shared mount in the other namespaces, that means any mounts under /var/named/chroot will be cloned to under /var/named/chroot/var/named/ in the other namespaces but not in the main namespace since it was set private.
