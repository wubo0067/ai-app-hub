## Environment

- Red Hat Enterprise Linux 7.8
- vxio Veritas modules 7.4.1.2100


## Issue

- Server panicked with hard lockup involving vxio veritas module.
- Server panic with this error Kernel panic - not syncing: Hard LOCKUP.


## Resolution

- Contact the Veritas module vendor for further investigation since the spinlock was acquired byvxiomodule, which belong to Veritas.

`vxio`
## Root Cause

- "Kernel panic - not syncing: Hard LOCKUP"  occurred due to spinlock was acquired by vxio module.
- Server panicked due to a hard lockup on a CPU. A hard lockup is a situation when a CPU stops responding to interrupts for a set period of time.
