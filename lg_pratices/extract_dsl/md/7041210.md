## Resolution

- Check the VMware ESXi host/hypervisor log to ensure that the guest machine is not a victim of any other virtualization overhead. For example; VMotion, VMware host maintenance activity like a backup of its configuration, etc.
- Disable the sysctl parameter "softlockup_panic" to avoid a false-positive softlockup and reboot. It should be enabled only in case of troubleshooting a problem.


## Root Cause

- A soft lockup is a symptom that the kernel loops in kernel mode without giving other tasks a chance to run for more than 20 seconds.
- In this case, since the soft lockup happened in userspace, the hypervisor should be checked.
