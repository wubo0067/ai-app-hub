## Environment

- Red Hat Enterprise Linux 6.4
- Red Hat Enterprise Linux 5


## Issue

- After enabling NFS/RPC debugging, the system rebooted with no information regarding the crash within the/var/log/messagesfile.
- Server crashed need to know root cause.

`/var/log/messages`
## Resolution

- Disable RPC/NFS debugging. Do not enable this additional debugging if NFS is being highly utilized on the server.


## Root Cause

- By enabling debugging on a system that is at load capacity and highly utilizing NFS, the increase processing load that is introduced by enabling debugging causes the system to encounter hang condition that will likely result in a kernel panic.
