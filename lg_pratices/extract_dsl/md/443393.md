## Environment

- Red Hat Enterprise Linux 6.3
- Red Hat Enterprise Linux 6.4


## Issue

System was rebooted unexpectedly and vmcore was created automatically.


## Resolution

This issue involves a third party kernel module 'snumbd26' so we are unable to diagnose it further.  Recommend raising a case with the vendor of this kernel module.


## Root Cause

The root cause of this panic is due to a task disabling interrupts for too long while trying to acquire a spinlock and a watchdog has detected this and considered the system to be locked up.


## Diagnostic Steps

- This is quite a bit long status locked by snumbd26. so watchdog detected and system made crash.
