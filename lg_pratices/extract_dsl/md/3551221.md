## Environment

- Red Hat Enterprise Linux
- Veritas modules


## Issue

- Server panicked with hard lockup involving veritas modules
- Server panic with this errorKernel panic - not syncing: Hard LOCKUP

`Kernel panic - not syncing: Hard LOCKUP`
## Resolution

- Contact theVeritas module vendorfor further investigation since the panic task and most of the hung state processes are stuck while processing those modules.

`Veritas module vendor`
## Root Cause

- Server panicked due to a hard lockup on a particular CPU. A hard lockup is a situation when a CPU stops responding to interrupts for a set period of time.
- This happened due to severe spinlock contention on a request queue for a block device andvxfsmodule found to be the owner of the lock.
