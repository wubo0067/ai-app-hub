## Root Cause

- A hard lockup is defined as a bug that causes a CPU to loop in kernel mode without giving other tasks a chance to run for more than the seconds defined with a sysctl knob, "watchdog_thresh", which is 60 seconds by default in RHEL6.
- The hard lockup was detected on a CPU on which a task was stuck waiting for a tasklist_lock rwlock_t lock with interrupts being disabled.
- At that time, another task had taken that tasklist_lock in PF hander code path and then was writing OOM messages to the serial console while performing oom-killing where there was a high amount of memory pressure.
- Since writing lots of messages to the serial console was very slow and time consuming job, the hard lockup occurred on the CPU that was waiting for the  tasklist_lock held by the task writing OOM messages to the serial console.
