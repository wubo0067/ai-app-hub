## Resolution

- It is recommended to open a support case with the hardware vendor and run a complete hardware diagnostic to find and replace the faulty CPU.
- Upgrade the BIOS firmware to the latest version.
- If you are facing the similar issue on a VM running on VMware hypervisor then the most likely cause would be the hypervisor malfunction or the underlying hardware glitch.


## Root Cause

- The hard-lockup was detected because the last saved value of the CPU-specific interrupt counter (hrtimer_interrupts_saved) and the current value of the CPU-specific interrupts counter (hrtimer_interrupts) are equal during a periodic check.
