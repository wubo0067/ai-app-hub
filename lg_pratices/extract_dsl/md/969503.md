## Environment

- Red Hat Enterprise Linux 6.5
- HP DL580 G8 server
- HP Blade BL460c Gen9
- hp-health package installed


## Issue

After a fresh RHEL 6.5 install on a HP DL580 gen8, system keeps crashing


```
[root@server123 crash]# grep -A1 "ernel panic" */vmcore-dmesg.txt 
<0>Kernel panic - not syncing: Watchdog detected hard LOCKUP on cpu 76
<4>Pid: 13955, comm: IrqRouteTbl Not tainted 2.6.32-431.11.2.el6.x86_64 #1
<0>Kernel panic - not syncing: Watchdog detected hard LOCKUP on cpu 15
<4>Pid: 11579, comm: IrqRouteTbl Not tainted 2.6.32-431.11.2.el6.x86_64 #1
<0>Kernel panic - not syncing: Watchdog detected hard LOCKUP on cpu 30
<4>Pid: 11583, comm: IrqRouteTbl Not tainted 2.6.32-431.11.2.el6.x86_64 #1
```

## Resolution

- Uninstall hp-health package or update to hp-health-10.00-1688.34.rhel6.x86_64


## Root Cause

"IrqRouteTbl" process is part of "hp-health" package.


## Diagnostic Steps

- Checkvmcore'sring buffer.
