## Root Cause

- A few timer subsystems have been implemented at private bug 1568294 in RHEL 7.6.
- The Hard LOCKUP was detected in this Hard LOCKUP detector code.Rawkernel/watchdog.c

 287 #ifdef CONFIG_HARDLOCKUP_DETECTOR
 288 /* watchdog detector functions */
 289 static bool is_hardlockup(void)
 290 {
 291         unsigned long hrint = __this_cpu_read(hrtimer_interrupts);
 292 
 293         if (__this_cpu_read(hrtimer_interrupts_saved) == hrint)
 294                 return true;
 295 
 296         __this_cpu_write(hrtimer_interrupts_saved, hrint);
 297         return false;
 298 }
 299 #endif
- hrtimer_interrupts is equal to hrtimer_interrupts_saved on CPU 7; which resulted in the crash.Rawcrash> p hrtimer_interrupts:7
per_cpu(hrtimer_interrupts, 7) = $1 = 884483

crash> p hrtimer_interrupts_saved:7
per_cpu(hrtimer_interrupts_saved, 7) = $2 = 884483
- However CPU 7 wasn't actually locked up. This means the Hard LOCKUP was false-positive.
