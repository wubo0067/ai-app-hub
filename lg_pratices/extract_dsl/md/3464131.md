## Resolution

- In RHEL 7.4 the HPSA driver was making a call with timeout; however, it should not have timeout.  This was changed in driver rebase in RHEL 7.7 GA kernel.


```
in kernel 3.10.0-693.11.6

  rc = hpsa_scsi_do_simple_cmd(h, c, reply_queue, DEFAULT_TIMEOUT);
                                                  ^^^^^^^^^^^^^^^

versus kernel 3.10.0-1062

  rc = hpsa_scsi_do_simple_cmd(h, c, reply_queue, NO_TIMEOUT);                        
                                                  ^^^^^^^^^^
```

## Root Cause

- HPSA's timeout handling inhpsa_scsi_do_simple_cmd_core()is unsafe.  In order for it to support a timeout, it must clean up a completion structure  and not leave it on the stack still referenced after timing out.
- hpsa_scsi_do_simple_cmd_core()would be hpsa's function which should have waited for the completion.  Buthpsa_scsi_do_simple_cmd_core()can time out and its timeout handling is unsafe.  It useswait_for_completion_io_timeout()to wait for a completion on the stack, then exits if a timeout occurs without doing any cleanup or prevention of the completion on the stack being used after the function returns.
