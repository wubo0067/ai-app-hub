- The cause is currently unknown. In some instances, replacement of the memory modules correlated with the issue no longer occurring. However, the causality is unknown.
- It's often quite difficult for Red Hat to show some hard evidence of faulty hardware (e.g. CPUs, DIMM modules, etc.) from the vmcore analysis. We can only suggest it's likely when hard lockups occur on CPUs that are idle or are just running userspace code with interrupts enabled.
