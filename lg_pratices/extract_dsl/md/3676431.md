## Resolution

- This is addressed via publicly available ErrataRHBA-2018:3763errata. So is fixed  in kernel versions kernel-2.6.32-754.9.1.el6 and later.
- This problem was tracked in private Red HatBugzilla #1624747.


## Workaround

- Use the kernel version2.6.32-754.3.5.el6or earlier.

`2.6.32-754.3.5.el6`
## Root Cause

- This is a regression introduced by the following patch in the kernel version2.6.32-754.6.3.el6.  This patch is associated withBZ# 1574568.

`2.6.32-754.6.3.el6``BZ# 1574568`
```
block: Let blk_drain_queue() caller obtain the queue lock

        Let the caller of blk_drain_queue() obtain the queue lock to improve
        readability of the patch called "Avoid that request_fn is invoked on
        a dead queue".
```

`block: Let blk_drain_queue() caller obtain the queue lock

        Let the caller of blk_drain_queue() obtain the queue lock to improve
        readability of the patch called "Avoid that request_fn is invoked on
        a dead queue".`
