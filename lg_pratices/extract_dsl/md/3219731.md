## Root Cause

- There was a bug inlpfcmodule's ELS (Extended Link Services) processing path for sli4 Emulex HBAs,  due to which it was usinghbalockinstead ofring_lockfor removing the elements fromtxcmplqlist.  Due to the use of incorrect lock,txcmplqlist was not protected from simultaneous updates and got corrupt. Processing this corrupttxcmplqlist (lpfc_iocbq) was leading to a hard lockup and crash.
- As the part of fix for above bug, driver will now first verify sli revision of the HBA and then would select a right lock to be used duringtxcmplqlist processing.
