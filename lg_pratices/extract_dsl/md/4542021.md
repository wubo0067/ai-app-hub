## Environment

- Red Hat Enterprise Linux 7.5 - 7.7
- Microsoft Hyper-V or Azure


## Issue

- Bad page stateorBad page map in processorBad rss-counter statereported in kernel log
- shortly after the above event VM hangs or is being crashed by soft lockup/hard lockup watchdog or hunged task detector

`Bad page state``Bad page map in process``Bad rss-counter state`
## Resolution

A  bug in kernel has been identified and it has been fixed inRed Hat Enterprise Linux 7.8. Please upgrade tokernel-3.10.0-1127.el7fromErrata RHSA-2020:1016.

`kernel-3.10.0-1127.el7`For previous  Red Hat Enterprise Linux releases:


#### Red Hat Enterprise Linux 7.7

- Upgrade tokernel-3.10.0-1062.18.1.el7fromErrata RHSA-2020:0834.

`kernel-3.10.0-1062.18.1.el7`
#### Red Hat Enterprise Linux 7.6 (EUS)

- Upgrade tokernel-3.10.0-957.46.1.el7fromErrata RHSA-2020:0664.

`kernel-3.10.0-957.46.1.el7`
#### Red Hat Enterprise Linux 7.5 (EUS)

- Upgrade tokernel-3.10.0-862.48.1.el7fromErrata RHSA-2020:0543.

`kernel-3.10.0-862.48.1.el7`The followingworkaroundsare available:

- disable Transparent Huge Pages
- if the issue is triggered byomicliprocess then reach Microsoft for update of OMS Agent. Version 1.12.15 has a workaround for the bug found in kernel

`omicli`
## Root Cause

The root cause of the issue is a problem with serialization betweengup_fast()function andpmdp_splitting_flush().

`gup_fast()``pmdp_splitting_flush()`
