## 🎯 最终诊断结果

**崩溃类型**: Kernel pointer corruption leading to page fault in idle loop

**Panic 信息**: BUG: unable to handle kernel paging request at 0000000065db75c7

**故障指令**: RIP: ffffffff8656bf75 (pause) ; actual fault from earlier load: mov (%rax),%rax at ffffffff8656bf6c

**根本原因**: A kernel pointer (likely a task_struct field) was overwritten with physical address 0x65db75c7, which is a reserved page, due to stray DMA write in IOMMU Passthrough mode. The corrupted pointer was dereferenced in cpu_idle_poll.

**详细分析**:

The crash occurred in cpu_idle_poll on CPU 7 (swapper task). The RIP points to a pause instruction, which cannot fault, indicating a RIP-CR2 contradiction. The actual faulting instruction is the preceding load: mov (%rax),%rax at offset +44. RAX held 0x65db75c7 at fault time (CR2).
