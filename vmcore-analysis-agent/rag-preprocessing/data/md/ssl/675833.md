## Resolution

- If this is an NFS server try reducing the nfsd task count.
- It's possible the soft lockup warnings are false alarms so try disabling the watchdog with:


```
# echo 0 > /proc/sys/kernel/watchdog
```

`# echo 0 > /proc/sys/kernel/watchdog`
## Root Cause

It appears that there were a lot of tasks waiting for page writeback to complete and the xfsdatad task had to wake all of them up while holding a spinlock with interrupts disabled and it took longer than expected and triggered the soft lockup watchdog to think the CPU was stuck.
