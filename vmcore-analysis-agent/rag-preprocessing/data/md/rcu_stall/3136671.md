## Root Cause

The system has found a race condition which can lead to a deadlock condition in the scheduler when one task tries to wake up an already running task. This race condition has resulted in one task spinning in a loop while waiting for a second task to stop running. But the second task cannot stop running as it is spinning waiting on a spinlock owned by the first task. This then resulted in NIC firmware hang.
