## Resolution

- Theoracleafdunsigned (U) kernel module is accessing the slab object which does not belong to it.
- Engageoracleafdmodule vendor for further assistance on this issue.

Theoracleafdunsigned (U) kernel module is accessing the slab object which does not belong to it.

`oracleafd`Engageoracleafdmodule vendor for further assistance on this issue.

`oracleafd`
## Root Cause

- The SLAB object is at least partially overwritten with the "oracleafd"-related signature (the "TSQR" ASCII string and the "AFDopenHndl" object reference), which may be a race condition of some type where the module uses the SLAB object which doesn't belong to it.
