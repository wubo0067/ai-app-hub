## Resolution

- Open a case with the provider of a third-party module [strmfs_mod] and take their opinion on this issue.
- Check for any known issue in [strmfs_mod] module & any patches or updates available to resolve this issue.
- Blacklist the unsigned (U) module [strmfs_mod] and check if this issue reproducible.


## Root Cause

- Thestrm11task was trying to acquire the spinlock inside the functiondupb().
- However, the function dupb() is part of a third party [strmfs_mod] kernel module.
