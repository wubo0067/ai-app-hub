## Environment

- RHEL 7
- kernel prints messages to slow serial console


## Issue

Message "IPVS: __ip_vs_del_service: enter" printed in kernel log buffer at a very high rate which causes different kinds of lockups and system hang.


## Resolution

Upgrade to kernel-3.10.0-1160.el7 (or newer) released viaRHSA-2020:4060as part of RHEL 7.9 release. Fix per kernel RPM's changelog:


```
- [net] ipvs: Remove noisy debug print from ip_vs_del_service (Alexey Klimov) [1769816]
```

`- [net] ipvs: Remove noisy debug print from ip_vs_del_service (Alexey Klimov) [1769816]`
## Root Cause

The issue is similar to one described here:Excessive printing to console may lead to logging of messages "rcu_sched detected stalls on CPUs/tasks"
